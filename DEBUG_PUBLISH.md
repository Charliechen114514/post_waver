# 🔍 注入模板发布调试指南

## 📋 完整测试流程

### 1️⃣ 准备工作

```bash
# 1. 确保服务器已重启（使用新代码）
pnpm stop:dev
pnpm dev

# 2. 运行测试脚本验证功能
npx tsx scripts/test-publish-with-template.ts
```

### 2️⃣ 前端操作步骤

1. **扫描文章**
   - 访问 http://localhost:5173/post_waver/publish
   - 点击"🔄 扫描文章"

2. **创建注入模板**
   - 访问 http://localhost:5173/post_waver/templates
   - 点击"➕ 新建模板"
   - 填写内容并保存

3. **为文章选择模板**
   - 回到发布页面
   - 选择一篇文章
   - **重要**：在"注入内容"下拉框中选择一个模板
   - 观察：预览应该显示注入的内容

4. **加入发布列表**
   - 勾选"加入发布列表"
   - 选择另一篇文章，选择不同的模板
   - 勾选"加入发布列表"

5. **批量发布**
   - 点击"📦 批量发布"
   - 确认发布

### 3️⃣ 验证输出

发布完成后，检查生成的文件：

```bash
# 查看最新的发布目录
ls -lt output/ | head -5

# 查看掘金文件（应该包含注入内容）
cat output/[最新目录]/juejin.txt

# 查看微信文件（应该包含注入内容和相关链接）
cat output/[最新目录]/wechat.txt
```

## 🐛 调试技巧

### 检查请求数据

打开浏览器开发者工具（F12）→ Network标签，查看批量发布请求：

```json
{
  "postIds": ["post1", "post2"],
  "skipPreview": true,
  "templateMap": {
    "post1": "template-id-1",
    "post2": "template-id-2"
  },
  "includeRelatedLinks": true
}
```

**关键点**：`templateMap` 应该包含文章ID到模板ID的映射。

### 检查服务器日志

服务器日志应该显示：

```
✅ 找到模板: 订阅引导
✅ 已应用注入模板: 订阅引导
✅ 已应用相关链接
```

如果看到这些日志，说明功能正常工作。

## ⚠️  常见问题

### 问题1: 输出文件没有注入内容

**可能原因**：
- 没有选择模板直接发布
- 服务器使用了旧代码
- 模板未启用

**解决方案**：
1. 确保在发布前选择了模板
2. 重启服务器：`pnpm stop:dev && pnpm dev`
3. 检查模板是否启用

### 问题2: 预览有注入，但输出没有

**原因**：预览和发布使用不同的代码路径

**解决方案**：
- 确保批量发布时 `templateMap` 正确传递
- 检查服务器日志确认应用了注入

### 问题3: 模板选择不生效

**原因**：可能切换文章后模板被重置

**解决方案**：
- 每选择一篇文章后，重新选择模板
- 确保勾选了"加入发布列表"
