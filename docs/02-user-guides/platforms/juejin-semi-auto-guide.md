## 一、编辑器概述

掘金使用的是**ByteMD**编辑器，这是字节跳动开源的Markdown编辑器，具有以下特点：

- ✅ 支持标准的Markdown语法
- ✅ 支持GitHub Flavored Markdown (GFM)
- ✅ 插件化架构，可扩展
- ✅ 轻量级，性能优秀
- ✅ 支持Vue/React/Svelte等框架

## 二、Markdown支持

### 2.1 完全支持的语法

#### 基础语法

| 语法 | 示例 | 说明 |
|------|------|------|
| 标题 | `# H1` ~ `###### H6` | 支持1-6级标题 |
| 粗体 | `**粗体**` | 双星号包围 |
| 斜体 | `*斜体*` | 单星号包围 |
| 粗斜体 | `***粗斜体***` | 三星号包围 |
| 删除线 | `~~删除线~~` | 双波浪线包围 |
| 行内代码 | `` `代码` `` | 反引号包围 |
| 代码块 | <code>```language</code> | 三个反引号，可指定语言 |
| 分割线 | `***` 或 `---` | 三个或更多星号/短横线 |

#### 列表

```markdown
# 无序列表
- 项目1
- 项目2
  - 嵌套项目

# 有序列表
1. 项目1
2. 项目2

# 任务列表
- [x] 已完成
- [ ] 未完成
```

#### 引用

```markdown
> 这是引用文本
> 可以有多行
```

#### 链接和图片

```markdown
# 链接
[掘金](https://juejin.cn)
[带标题](https://juejin.cn "标题")

# 图片
![图片描述](https://example.com/image.jpg)
```

#### 表格（GFM）

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |

# 带对齐
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| A      |  B   |      C |
```

### 2.2 扩展语法支持

#### 数学公式（KaTeX）

掘金通过ByteMD插件支持LaTeX数学公式：

**行内公式**：
```markdown
$E = mc^2$
```

**块级公式**：
```markdown
$$
\frac{n!}{k!(n-k)!} = \binom{n}{k}
$$
```

**常用符号**：
- 上标：`x^2`
- 下标：`H_2O`
- 分数：`\frac{a}{b}`
- 根号：`\sqrt{x}`
- 求和：`\sum_{i=1}^{n}`
- 积分：`\int_{a}^{b}`

#### 流程图（Mermaid）

```markdown
\```mermaid
graph TD
    A[开始] --> B{判断}
    B -->|是| C[操作1]
    B -->|否| D[操作2]
    C --> E[结束]
    D --> E
\```
```

## 三、HTML支持

### 3.1 支持的HTML标签

掘金的ByteMD编辑器会**过滤或转换部分HTML标签**，以确保安全性。

**基本支持**（需测试确认）：
- `<div>` - 块级容器
- `<span>` - 行内容器
- `<p>` - 段落
- `<br>` - 换行
- `<a>` - 链接
- `<img>` - 图片

**可能被过滤**：
- `<script>` - 脚本标签（安全原因）
- `<iframe>` - 内联框架（安全原因）
- `<style>` - 样式标签（可能被移除）
- 带有`on*`事件属性的标签（安全原因）

### 3.2 样式处理

**内联样式**：
- ⚠️ **基本不支持**：掘金主要通过Markdown语法控制样式
- 部分样式可能被保留，但不推荐依赖

**CSS类**：
- ❌ **不支持**：无法添加自定义class或id

**最佳实践**：
- 优先使用纯Markdown语法
- 避免使用HTML标签控制样式
- 如果需要特殊效果，考虑使用Mermaid或数学公式插件

## 四、图片处理

### 4.1 外链图片

✅ **完全支持**外链图片

**推荐做法**：
```markdown
![图片描述](https://example.com/image.jpg)
```

**支持的图片格式**：
- JPG / JPEG
- PNG
- GIF（动图也支持）
- WebP（需测试）
- SVG（需测试）

### 4.2 本地图片

✅ **支持拖拽上传**

**操作方式**：
1. 在编辑器中直接拖拽图片文件
2. 图片会自动上传到掘金CDN
3. 自动生成Markdown语法

**图片大小限制**：
- 最大文件大小：**5MB**（需确认）
- 推荐尺寸：宽度不超过 **1920px**

### 4.3 图片对齐和尺寸

**基本语法**：
```markdown
![图片](url)
```

**尺寸控制**：
- 掘金会自动调整图片大小以适应屏幕
- 暂不支持通过HTML属性控制宽高
- 推荐在上传前调整好图片尺寸

**对齐方式**：
- 默认居中显示
- 暂不支持左/右对齐

## 五、粘贴行为

### 5.1 从其他Markdown编辑器粘贴

✅ **支持良好**

**推荐编辑器**：
- Typora
- VSCode + Markdown插件
- Obsidian
- Mark Text

**行为**：
- 格式会被保留
- 代码块语言标识会被保留
- 图片需要重新上传（除非是外链）

### 5.2 从富文本编辑器粘贴

⚠️ **可能丢失格式**

**来源**：
- Word文档
- 网页内容
- 富文本编辑器

**建议**：
- 先转换为Markdown格式
- 使用转换工具（如Pandoc）
- 手动调整格式

### 5.3 从网页复制粘贴

⚠️ **需要清理格式**

**行为**：
- 可能带有多余的HTML标签
- 链接可能被保留
- 图片需要重新上传

**建议工具**：
- 使用"粘贴为纯文本"功能
- 使用格式清理工具
- 手动转换为Markdown

## 七、最佳实践

### 7.1 推荐的转换策略

对于Content Hub项目，掘金平台的转换策略：

```typescript
// JuejinConverter 转换流程
1. 解析Markdown源文件
2. 验证语法兼容性（确保所有语法都被掘金支持）
3. 处理特殊元素：
   - 数学公式：保留LaTeX格式（掘金支持）
   - Mermaid图表：保留语法（掘金支持）
   - 代码块：确保语言标识正确
4. 处理图片：
   - 本地图片：提示用户手动上传
   - 外链图片：直接保留
5. 输出：标准Markdown格式
```

### 7.2 需要特殊处理的元素

#### 数学公式

**源格式**：
```markdown
$E = mc^2$
```

**掘金支持**：✅ 完全支持，无需转换

#### 脚注

**源格式**：
```markdown
这是脚注[^1]

[^1]: 脚注内容
```

**掘金支持**：⚠️ 需测试
- 如果不支持：转换为普通文本或删除
- 如果支持：直接保留

#### 任务列表

**源格式**：
```markdown
- [x] 已完成
- [ ] 未完成
```

**掘金支持**：✅ 完全支持（GFM语法）

### 7.3 注意事项

1. **避免使用HTML控制样式**
   - 掘金主要通过Markdown语法渲染
   - HTML标签可能被过滤

2. **代码块必须指定语言**
   ```markdown
   // ❌ 不推荐
   ```
   code here
   ```

   // ✅ 推荐
   ```javascript
   code here
   ```
   ```

3. **表格使用GFM语法**
   - 不要使用HTML表格
   - 使用管道符`|`格式

4. **图片使用外链或手动上传**
   - Content Hub可以提供外链图片
   - 本地图片需要用户手动上传到掘金

5. **数学公式使用LaTeX**
   - 不要使用图片代替公式
   - 使用标准LaTeX语法

## 八、常见问题

### Q1: 粘贴后格式乱了怎么办？

**A**: 推荐做法：
1. 使用纯Markdown编辑器（如Typora）复制
2. 使用"粘贴为纯文本"功能
3. 手动调整格式

### Q2: 代码块没有高亮？

**A**: 检查：
1. 是否指定了语言标识
2. 语言名称是否正确
3. 使用小写（推荐）

### Q3: 表格显示不正常？

**A**: 确保：
1. 使用GFM表格语法（管道符）
2. 表头分隔符至少3个短横线
3. 列对齐使用冒号（可选）

### Q4: 数学公式不显示？

**A**: 检查：
1. 使用单个`$`表示行内公式
2. 使用双个`$$`表示块级公式
3. LaTeX语法是否正确

### Q5: 本地图片怎么处理？

**A**:
1. 在掘金编辑器中拖拽上传
2. 或使用图床服务生成外链
3. Content Hub可以集成图床上传功能
