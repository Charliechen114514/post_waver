> **版本**: v1.0

## 🚀 快速开始

### 3分钟发布到知乎

1. **转换内容**: 使用PostWaver将Markdown转换为知乎格式
   ```bash
   pnpm transform:zhihu content/posts/your-article.md
   ```

2. **复制内容**: 点击"复制到剪贴板"按钮

3. **粘贴到知乎**: 
   - 访问 https://zhuanlan.zhihu.com/write
   - 粘贴内容到编辑器
   - 检查格式和图片占位符

4. **上传图片**: 根据`📷 图片: xxx (请手动上传)`提示上传本地图片

5. **发布**: 设置标题、话题后发布

## 📖 完整指南

### 一、知乎编辑器概述

知乎使用的是**Markdown编辑器**，具有以下特点：

- ✅ 支持标准的Markdown语法
- ✅ 支持GitHub Flavored Markdown (GFM)
- ✅ 支持LaTeX数学公式
- ✅ 支持代码高亮
- ✅ 支持表格、任务列表等GFM语法
- ✅ 支持本地图片拖拽上传
- ✅ 支持丰富的编辑工具栏

---

### 二、Markdown支持

#### 2.1 完全支持的语法

**基础语法**

| 语法 | 示例 | 说明 |
|------|------|------|
| 标题 | `# H1` ~ `###### H6` | 支持1-6级标题 |
| 粗体 | `**粗体**` | 双星号包围 |
| 斜体 | `*斜体*` | 单星号包围 |
| 粗斜体 | `***粗斜体***` | 三星号包围 |
| 删除线 | `~~删除线~~` | 双波浪线包围 |
| 行内代码 | `` `代码` `` | 反引号包围 |
| 代码块 | <code>```language</code> | 三个反引号，可指定语言 |
| 分割线 | `***` 或 `---` | 三个或更多星号/短横线 |

**列表**

```markdown
# 无序列表
- 项目1
- 项目2
  - 嵌套项目

# 有序列表
1. 项目1
2. 项目2

# 任务列表
- [x] 已完成
- [ ] 未完成
```

**引用**

```markdown
> 这是引用文本
> 可以有多行
```

**链接和图片**

```markdown
# 链接
[知乎](https://www.zhihu.com)
[带标题](https://www.zhihu.com "标题")

# 图片
![图片描述](https://example.com/image.jpg)
```

**表格（GFM）**

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |

# 带对齐
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| A      |  B   |      C |
```

---

#### 2.2 扩展语法支持

**数学公式（LaTeX）**

知乎支持LaTeX数学公式：

**行内公式**：
```markdown
$E = mc^2$
```

**块级公式**：
```markdown
$$
\frac{n!}{k!(n-k)!} = \binom{n}{k}
$$
```

**常用符号**：
- 上标：`x^2`
- 下标：`H_2O`
- 分数：`\frac{a}{b}`
- 根号：`\sqrt{x}`
- 求和：`\sum_{i=1}^{n}`
- 积分：`\int_{a}^{b}`

**代码块与高亮**：
知乎提供了丰富的代码高亮支持，包括大多数主流编程语言。

### 三、HTML支持

#### 3.1 支持的HTML标签

知乎编辑器会**过滤或转换部分HTML标签**，以确保安全性。

**基本支持**（需测试确认）：
- `<div>` - 块级容器
- `<span>` - 行内容器
- `<p>` - 段落
- `<br>` - 换行
- `<a>` - 链接
- `<img>` - 图片

**可能被过滤**：
- `<script>` - 脚本标签（安全原因）
- `<iframe>` - 内联框架（安全原因）
- `<style>` - 样式标签（可能被移除）
- 带有`on*`事件属性的标签（安全原因）

#### 3.2 样式处理

**内联样式**：
- ⚠️ **基本不支持**：知乎主要通过Markdown语法控制样式
- 部分样式可能被保留，但不推荐依赖

**CSS类**：
- ❌ **不支持**：无法添加自定义class或id

**最佳实践**：
- 优先使用纯Markdown语法
- 避免使用HTML标签控制样式
- 如果需要特殊效果，考虑使用数学公式或代码块

### 四、图片处理

#### 4.1 外链图片

✅ **完全支持**外链图片

**推荐做法**：
```markdown
![图片描述](https://example.com/image.jpg)
```

**支持的图片格式**：
- JPG / JPEG
- PNG
- GIF（动图也支持）
- WebP（需测试）
- SVG（需测试）

#### 4.2 本地图片

✅ **支持拖拽上传**

**操作方式**：
1. 在编辑器中直接拖拽图片文件
2. 图片会自动上传到知乎图床
3. 自动生成Markdown语法

**图片大小限制**：
- 最大文件大小：**5MB**（需确认）
- 推荐尺寸：宽度不超过 **1920px**

#### 4.3 图片对齐和尺寸

**基本语法**：
```markdown
![图片](url)
```

**尺寸控制**：
- 知乎会自动调整图片大小以适应屏幕
- 暂不支持通过HTML属性控制宽高
- 推荐在上传前调整好图片尺寸

**对齐方式**：
- 默认居中显示
- 暂不支持左/右对齐

### 五、PostWaver知乎转换器

#### 5.1 转换器功能

PostWaver的知乎转换器位于 `packages/transformer/src/to-zhihu.ts`，提供以下功能：

**核心功能**：
- ✅ 保持原始Markdown格式
- ✅ 处理本地图片（转换为上传占位符）
- ✅ 支持相关文章链接生成
- ✅ 支持仓库引用注入

#### 5.2 使用方法

**命令行使用**：
```bash
# 转换单个文件
pnpm transform:zhihu content/posts/your-article.md

# 转换并保存
pnpm transform:zhihu content/posts/your-article.md > output/zhihu-article.md
```

**编程使用**：
```typescript
import { transformForZhihu } from '@content-hub/transformer'

const markdown = '# 标题\n\n这是内容'
const result = await transformForZhihu(markdown, {
  removeLocalImages: true,
  includeRelatedLinks: true,
  postId: 'post-123',
  relatedPosts: [...],
  blogBaseUrl: 'https://your-blog.github.io/'
})
```

#### 5.3 转换选项

```typescript
interface ZhihuTransformOptions {
  /** 文章ID */
  postId?: string
  /** 相关文章列表 */
  relatedPosts?: IndexedPost[]
  /** 博客基础URL */
  blogBaseUrl?: string
  /** 是否添加相关文章链接 */
  includeRelatedLinks?: boolean
  /** 是否移除本地图片（用于复制到外部平台） */
  removeLocalImages?: boolean
}
```

#### 5.4 本地图片处理

当启用 `removeLocalImages: true` 时，转换器会：

1. **识别本地图片**：查找 `assets/` 目录下的图片
2. **生成占位符**：替换为 `📷 图片: filename (请手动上传)`
3. **保留外链图片**：HTTP/HTTPS开头的图片URL保持不变

**示例**：
```markdown
# 输入
![本地图片](assets/images/screenshot.png)

# 输出
📷 图片: screenshot.png (请手动上传)
```

### 七、最佳实践

#### 7.1 推荐的转换策略

对于PostWaver项目，知乎平台的转换策略：

```typescript
// 知乎转换流程
1. 解析Markdown源文件
2. 验证语法兼容性（确保所有语法都被知乎支持）
3. 处理特殊元素：
   - 数学公式：保留LaTeX格式（知乎支持）
   - 代码块：确保语言标识正确
   - 表格：确保使用GFM语法
4. 处理图片：
   - 本地图片：转换为上传占位符
   - 外链图片：直接保留
5. 输出：标准Markdown格式
```

#### 7.2 需要特殊处理的元素

**数学公式**

**源格式**：
```markdown
$E = mc^2$
```

**知乎支持**：✅ 完全支持，无需转换

**任务列表**

**源格式**：
```markdown
- [x] 已完成
- [ ] 未完成
```

**知乎支持**：✅ 完全支持（GFM语法）

#### 7.3 注意事项

1. **避免使用HTML控制样式**
   - 知乎主要通过Markdown语法渲染
   - HTML标签可能被过滤

2. **代码块必须指定语言**
   ```markdown
   // ❌ 不推荐
   ```
   code here
   ```

   // ✅ 推荐
   ```javascript
   code here
   ```
   ```

3. **表格使用GFM语法**
   - 不要使用HTML表格
   - 使用管道符`|`格式

4. **图片使用外链或手动上传**
   - PostWaver会标记本地图片
   - 需要在知乎编辑器中手动上传

5. **数学公式使用LaTeX**
   - 不要使用图片代替公式
   - 使用标准LaTeX语法

### 八、常见问题

### Q1: 粘贴后格式乱了怎么办？

**A**: 推荐做法：
1. 使用PostWaver转换后的内容
2. 检查是否启用了正确的转换选项
3. 手动调整小部分格式问题

### Q2: 代码块没有高亮？

**A**: 检查：
1. 是否指定了语言标识
2. 语言名称是否正确
3. 使用小写（推荐）

### Q3: 表格显示不正常？

**A**: 确保：
1. 使用GFM表格语法（管道符）
2. 表头分隔符至少3个短横线
3. 列对齐使用冒号（可选）

### Q4: 数学公式不显示？

**A**: 检查：
1. 使用单个`$`表示行内公式
2. 使用双个`$$`表示块级公式
3. LaTeX语法是否正确

### Q5: 本地图片怎么处理？

**A**:
1. PostWaver会生成上传占位符
2. 在知乎编辑器中拖拽上传
3. 或使用图床服务生成外链
4. 替换占位符文本

### Q6: 如何设置文章元数据？

**A**:
1. **标题**：在知乎编辑器顶部设置
2. **话题**：添加相关话题标签（知乎特有的分类方式）
3. **封面**：选择合适的封面图
4. **摘要**：自动生成或手动编辑

### Q7: 知乎文章如何排版更好？

**A**: 建议：
1. 使用清晰的标题层级
2. 适当使用引用块强调重点
3. 添加表格整理复杂信息
4. 使用代码块展示代码
5. 合理使用图片辅助说明

### 十、SEO优化

#### 10.1 话题选择

**选择合适的话题**：
- 话题要与文章内容高度相关
- 选择热度适中但竞争不过大的话题
- 可以添加2-5个话题标签

**话题权重**：
- 第一个话题权重最高
- 按重要性排序话题
- 避免添加无关话题

#### 10.2 标题优化

**标题建议**：
- 标题要包含核心关键词
- 长度控制在10-30字
- 避免使用过于夸张的标题
- 使用问句或数字可以提高点击率

#### 10.3 内容优化

**内容质量**：
- 提供有价值的内容
- 结构清晰，逻辑严密
- 适当使用图片和表格
- 添加实际案例和代码示例

**内链外链**：
- 适当引用知乎内部文章
- 可以添加权威外部链接
- 避免过度营销
