# M2.3 实施笔记

## 开发过程记录

### 关键决策

#### 1. 选择Hono.js作为Web框架
**决策**: 使用Hono.js而不是Express或Fastify

**理由**:
- 轻量级，启动速度快
- 现代化的API设计
- 良好的TypeScript支持
- 与现有技术栈兼容

**结果**: 服务器启动时间 < 1秒，内存占用 ~50MB

#### 2. 端口自动发现机制
**决策**: 实现端口自动发现（3737+），而不是固定端口

**理由**:
- 避免端口冲突
- 支持多个预览实例同时运行
- 提升用户体验

**实现**:
```typescript
async function tryPorts(startPort: number, maxAttempts: number = 10): Promise<number> {
  for (let i = 0; i < maxAttempts; i++) {
    const port = startPort + i
    // 尝试绑定端口
    // ...
  }
}
```

#### 3. URL编码处理
**问题**: 文件路径包含斜杠（`content/posts/test.md`），导致路由404

**解决方案**:
- 在CLI中使用 `encodeURIComponent()` 编码路径
- 在服务器端使用 `decodeURIComponent()` 解码
- 添加通配符路由作为fallback

**代码**:
```typescript
// CLI
const url = `http://localhost:${server.port}/preview/${platform}/${encodeURIComponent(file)}`

// Server
const decodedId = decodeURIComponent(id)
```

#### 4. 内联HTML模板
**决策**: 将HTML模板直接内联在TypeScript代码中，而不是使用独立的模板文件

**理由**:
- 简化部署（无需管理模板文件）
- 类型安全（TypeScript编译时检查）
- 减少I/O操作
- 便于单文件分发

**权衡**: 牺牲了模板的可编辑性，但提升了性能和简洁性

### 技术问题解决

#### 问题1: TypeScript编译错误 - `parsePost`未导出
**错误信息**:
```
error TS2305: Module '"@content-hub/core"' has no exported member 'parsePost'.
```

**原因**: `tsconfig.tsbuildinfo` 缓存导致类型定义未更新

**解决方案**:
```bash
rm -f tsconfig.tsbuildinfo
pnpm build
```

#### 问题2: Workspace依赖解析失败
**错误信息**:
```
Error [ERR_MODULE_NOT_FOUND]: Cannot find package '@content-hub/engine'
```

**原因**: `scripts/package.json` 中缺少 `@content-hub/engine` 依赖

**解决方案**: 添加依赖到 `scripts/package.json`
```json
{
  "dependencies": {
    "@content-hub/engine": "workspace:*"
  }
}
```

#### 问题3: Hono版本不兼容
**警告信息**:
```
✕ unmet peer hono@^4: found 3.12.12
```

**原因**: `@hono/node-server@^1.8.2` 需要 `hono@^4`

**解决方案**: 升级Hono到v4
```json
{
  "hono": "^4.0.0"  // 从 ^3.11.0 升级
}
```

#### 问题4: 浏览器API类型错误
**错误信息**:
```
error TS2304: Cannot find name 'window'.
error TS2304: Cannot find name 'document'.
```

**原因**: TypeScript配置中没有包含DOM类型，且代码在Node.js环境运行

**解决方案**: 简化clipboard.ts，移除浏览器特定代码，仅使用Node.js的clipboardy
